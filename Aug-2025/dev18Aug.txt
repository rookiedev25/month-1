Component communication using @Input and @Output:
-------------------------------------------------
1. Parent â†’ Child (@Input)

The parent passes data down to the child.

The property decorated with @Input() exists inside the child component, but the parent sets its value.

2. Child â†’ Parent (@Output)

The child emits an event up to the parent.

The property decorated with @Output() exists inside the child component, but the parent listens to it.

ðŸ”¹ Separation of Responsibilities

Child Component

Declares @Input() (to receive data).

Declares @Output() (to emit events).

Parent Component

Supplies data for @Input().

Listens to events from @Output().

child.component.ts
@Component({
  selector: 'app-child',
  template: `
    <p>Got from parent: {{ childMessage }}</p>
    <button (click)="sendNotification()">Notify Parent</button>
  `
})
export class ChildComponent {
  @Input() childMessage: string = '';     // For parent â†’ child
  @Output() notifyParent = new EventEmitter<string>(); // For child â†’ parent

  sendNotification() {
    this.notifyParent.emit("Hello Parent, I'm Child!");
  }
}

parent.component.html
<!-- Sending data to child + listening to event -->
<app-child 
  [childMessage]="'Hello Child from Parent'" 
  (notifyParent)="onNotify($event)">
</app-child>

parent.component.ts
export class ParentComponent {
  onNotify(message: string) {
    console.log("Received from child:", message);
  }
}
